# 报告结构配置功能 - 交互优化设计方案

## 1. 设计目标

### 1.1 核心问题
当前系统让业务人员面临三个困惑：
- **概念抽象**：需要理解"节点类型"（AI章节/固定内容/容器/数据分析）
- **顺序固定**：内容展示顺序无法调整（永远是表格→图表→解读）
- **内容死板**：固定文字无法自动变化（如"湖南省"需要手动改）

### 1.2 优化方向
让业务人员像**搭积木**一样配置报告：
- 拖拽调整内容顺序
- 勾选开关控制显示
- 插入变量自动替换

---

## 2. 整体架构变化

### 2.1 信息架构重构

**当前模型：**
```
结构 → 节点（4种类型）→ 配置项（题号/图表/表格/AI）
```
*问题：类型概念抽象，一个节点功能过载*

**优化后模型：**
```
结构 → 章节节点（仅标题层级） → 内容块（4种组件，可排序）
```
*优势：概念简单，一个章节 = 多个内容块按顺序排列*

### 2.2 内容块类型定义

| 内容块 | 业务含义 | 业务人员理解成本 |
|--------|----------|------------------|
| **固定引言** | 章节开头的说明文字 | ⭐ 极低（就是写文字） |
| **问题分析** | 数据表格 + AI分析解读 | ⭐ 极低（勾选题号即可） |
| **分析图表** | 可视化图表展示 | ⭐ 极低（勾选显示即可） |
| **AI深度解读** | 额外的AI生成段落 | ⭐ 低（写提示词或默认） |

## 3. 界面设计详解

### 3.1 布局结构

采用**三栏布局**，符合"左-中-右"操作习惯：

```
┌─────────────────┬────────────────────┬─────────────────────┐
│                 │                    │                     │
│   结构列表      │    结构层级树       │    配置面板         │
│   （左栏）      │    （中栏）         │    （右栏）         │
│                 │                    │                     │
│  患者用药调研   │    4. 问卷结果分析   │  【节点标题】       │
│  医生处方调研   │    ├─4.1 药品疗效   │  [输入框]           │
│                 │    │ ├─4.1.1 ◀──   │                     │
│  + 新建结构     │    │ └─4.1.2       │  【内容组件】       │
│                 │    ├─4.2 用药安全   │  ┌───────────────┐  │
│                 │    └─4.3 用药便利   │  │ ☰ 固定引言    │  │
│                 │                    │  ├───────────────┤  │
│                 │                    │  │ ☰ 问题分析    │  │
│                 │                    │  ├───────────────┤  │
│                 │                    │  │ ☰ 分析图表    │  │
│                 │                    │  └───────────────┘  │
│                 │                    │                     │
│                 │                    │  [+添加内容块]      │
│                 │                    │                     │
│                 │                    │  [保存配置]         │
└─────────────────┴────────────────────┴─────────────────────┘
  宽度: 260px      宽度: 自适应          宽度: 380px（固定）
```

### 3.2 左栏：结构列表

**样式描述：**

- **背景色**：纯白 (#FFFFFF)
- **宽度**：固定260px
- **分割线**：右侧1px浅灰色边框 (#E5E7EB)
- **列表项**：
  - 高度：72px
  - 内边距：16px
  - 悬停：背景色变为浅紫 (#F5F3FF)
  - 选中：左侧3px紫色边框 (#6366F1)，背景浅紫
- **新建按钮**：固定在底部，紫色主按钮

**交互逻辑：**
1. 点击结构名称 → 中栏加载该结构的章节树
2. 悬停显示编辑/删除图标（右侧）
3. 新建按钮点击 → 弹出新建结构抽屉（从右侧滑出）

### 3.3 中栏：结构层级树

**样式描述：**

- **背景色**：浅灰白 (#F9FAFB)
- **缩进规则**：
  - 一级节点（章）：无缩进，字号16px，字重600
  - 二级节点（节）：缩进24px，字号15px，字重500
  - 三级节点（点）：缩进48px，字号14px，字重400
- **节点样式**：
  - 高度：44px
  - 圆角：8px
  - 悬停：背景白色 + 阴影
  - 选中：白色背景 + 紫色左边框 + 阴影
- **拖拽指示器**：半透明占位条（高度40px，背景#6366F1，opacity 0.2）

**交互逻辑：**
1. 点击节点 → 右栏显示该节点配置
2. 拖拽节点 → 调整章节顺序（同级间）
3. 拖拽到节点上 → 变为该节点的子节点
4. 右键节点 → 弹出菜单（添加子节点/删除/重命名）

### 3.4 右栏：节点配置面板（核心）

**样式描述：**

- **背景色**：纯白 (#FFFFFF)
- **左侧边框**：1px浅灰色 (#E5E7EB) 与中栏分隔
- **内边距**：24px
- **标题输入框**：
  - 高度：44px
  - 圆角：8px
  - 边框：1px灰色，focus时紫色边框+阴影

#### 3.4.1 内容块卡片样式

每个内容块是一个**可拖拽卡片**：

```
┌──────────────────────────────────────────────────┐
│ ☰  固定引言                          [配置 ▼] [×] │
├──────────────────────────────────────────────────┤
│  内容区域                                           │
│  ┌────────────────────────────────────────────┐   │
│  │ 本次针对{place}地区{product}的调研显示...   │   │
│  │                                            │   │
│  │  变量工具栏：                              │   │
│  │  [{place}] [{time}] [{product}] [+更多]    │   │
│  └────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────┘
```

**卡片样式细节：**
- 背景：白色
- 边框：1px浅灰 (#E5E7EB)，hover时紫色 (#6366F1)
- 圆角：12px
- 阴影：无 → hover时轻微阴影 (0 2px 8px rgba(0,0,0,0.08))
- 拖拽手柄 (☰)：左侧灰色，hover变紫色
- 操作按钮：右侧灰色图标，hover变深

#### 3.4.2 四种内容块详细设计

**A. 固定引言块**

```
┌──────────────────────────────────────────────────┐
│ ☰  固定引言                                      │
├──────────────────────────────────────────────────┤
│                                                  │
│  文字内容：                                       │
│  ┌────────────────────────────────────────────┐ │
│  │ 本次针对{place}地区{product}的调研显示，共   │ │
│  │ 收集{sample_count}份有效问卷...              │ │
│  │                                            │ │
│  └────────────────────────────────────────────┘ │
│                                                  │
│  快捷插入变量：                                   │
│  [{place}] [{time}] [{product}] [{sample_count}] │
│                                                  │
└──────────────────────────────────────────────────┘
```

**变量按钮样式：**
- 背景：浅紫 (#F5F3FF)
- 文字：紫色 (#6366F1)
- 圆角：4px
- 悬停：深紫背景，白色文字
- 点击：插入到光标位置，格式为`{变量名}`

**B. 问题分析块**

```

**下拉选择器样式：**
- 高度：40px
- 圆角：8px
- 选项：Q1-Q50（根据Excel最大题号动态）
- 选中后显示题目标题（从Excel解析）

**C. 分析图表块**

```
┌──────────────────────────────────────────────────┐
│ ☰  分析图表                          [配置 ▼] [×] │
├──────────────────────────────────────────────────┤

│  [✓] 显示分析图表                                │

**提示词区域：**

- 占位符提示：支持`{dimension_title}` `{question_title}`变量

### 3.5 添加内容块交互

**触发位置：** 配置面板底部，紫色按钮

**展开菜单：**

```
[ + 添加内容块 ]
      │
      ▼
┌──────────────────────┐
│  📄 固定引言         │
│  📊 问题分析         │
│  📈 分析图表         │
│  🤖 AI深度解读       │
└──────────────────────┘
```

**菜单样式：**
- 宽度：180px
- 圆角：12px
- 阴影：较大 (0 8px 24px rgba(0,0,0,0.12))
- 图标：左侧彩色图标
- 悬停：背景浅紫，文字紫色

**添加后的动画：**
- 新块从底部滑入（300ms ease-out）
- 其他块自动下移
- 新块高亮闪烁（紫色边框1秒后恢复）

### 3.6 拖拽排序交互

**触发方式：**
- 鼠标按住内容块左侧 ☰ 图标
- 或 触摸屏长按内容块

**拖拽状态：**
- 原位置：半透明占位（opacity 0.4）
- 拖拽中：完整卡片跟随鼠标，透明度0.9
- 可放置位置：显示虚线指示条（紫色，2px高）

**释放行为：**
- 有效位置：卡片滑入新位置（200ms ease）
- 无效位置：卡片弹回原位置（弹性动画）

## 4. 关键交互流程

### 4.1 完整配置流程示例

**场景：** 配置"4.1.1 消化功能改善效果分析"章节

**Step 1: 选中节点**
- 业务人员在中栏点击"4.1.1"
- 右栏加载配置面板
- 显示默认已有：固定引言 + 问题分析

**Step 2: 编辑固定引言**
- 点击固定引言卡片的内容区域
- 输入："本次针对{place}地区"
- 光标停在{place}后，点击上方[{place}]按钮
- 变量高亮显示（紫色背景）

**Step 3: 调整问题分析**
- 点击问题分析的"配置"按钮
- 下拉选择：题号 Q1（治疗效果满意度）
- 确认两个勾选框都已勾选

**Step 4: 添加分析图表**
- 点击底部[ + 添加内容块]
- 选择"分析图表"
- 新卡片出现在问题分析下方
- 选择关联题号：Q1
- 勾选"显示分析图表"

**Step 5: 调整顺序**
- 拖拽"分析图表"卡片，放到"问题分析"上方
- 释放后顺序变为：固定引言 → 分析图表 → 问题分析

**Step 6: 保存配置**
- 点击[保存配置]按钮
- 按钮变为"保存中..."（旋转图标）
- 成功后显示绿色对勾，1秒后恢复原状

### 4.2 变量使用流程

**场景：** 在固定引言中插入变量

**操作步骤：**
1. 点击固定引言的内容输入框
2. 在需要插入变量的位置点击
3. 点击下方变量按钮（如[{place}]）
4. 变量以紫色标签形式插入文本中
5. 保存后，生成报告时自动替换为实际值

**变量标签样式：**
```
本次针对 {place} 地区 {product} 的调研显示...
      └─ 紫色圆角标签，可点击删除
```

## 5. 空状态与异常处理

### 5.1 空状态设计

**新建章节节点（无内容块）：**

```
┌──────────────────────────────────────────────────┐
│                                                  │
│           ┌──────────────┐                       │
│           │   📦 空箱子   │                       │
│           └──────────────┘                       │
│                                                  │
│         该章节暂无内容组件                        │
│                                                  │
│      点击"添加内容块"开始配置                     │
│                                                  │
│           [ + 添加内容块 ]                        │
│                                                  │
└──────────────────────────────────────────────────┘
```

### 5.2 删除确认

**删除内容块：**
- 悬停显示删除图标（右上角 ×）
- 点击后卡片抖动，显示确认按钮
- 确认后卡片向左滑出并删除（300ms）

**删除章节节点：**
- 右键菜单选择"删除"
- 弹窗确认："确定删除"4.1.1"及其所有内容？此操作不可撤销。"
- 确认后节点从中栏移除，下级节点自动提升

### 5.3 错误提示

**未保存离开：**
- 尝试切换节点或关闭页面时
- 弹出抽屉提示："您有未保存的更改"
- 选项：[丢弃更改] [保存并离开] [取消]

**必填项缺失：**
- 题号未选择：下拉框边框变红，下方红色提示"请选择题号"
- 保存时校验，阻止保存并滚动到第一个错误

## 6. 响应式适配

### 6.1 大屏（≥1440px）
- 三栏完整显示
- 右栏宽度380px（固定，不随屏幕变大）

### 6.2 中屏（1024px-1439px）
- 左栏收缩为图标模式（仅显示结构首字）
- 中栏自适应
- 右栏保持380px

### 6.3 小屏（<1024px）
- 左栏隐藏，通过顶部导航切换结构
- 中栏和右栏变为Tab切换（层级树 | 配置）
- 配置面板全屏显示

## 7. 与现有系统的对比优势

| 维度 | 现有系统 | 优化后系统 |
|------|----------|------------|
| **概念复杂度** | 需理解4种节点类型 | 只需理解"内容块" |
| **操作步骤** | 6-8步完成配置 | 3-5步完成配置 |
| **灵活度** | 顺序固定 | 拖拽即可调整 |
| **内容动态性** | 纯静态文字 | 支持变量自动替换 |
| **学习成本** | 高（需培训） | 低（直觉操作） |
| **错误率** | 中（类型选错） | 低（勾选即可） |
